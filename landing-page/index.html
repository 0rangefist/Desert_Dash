About

I have always been fascinated by video games since I was a kid. I loved the thrill of exploring different worlds and overcoming challenges. As a software engineer, I wanted to create my own game and share it with others. That's why I decided to take on this project as part of my development at Holberton School.

I chose to make a desert-themed game because I was inspired by one of my favorite games, Uncharted 3. I was amazed by the realistic graphics and the immersive story of Nathan Drake's adventure in the Arabian desert. I wanted to capture some of that essence in my own game, but with a simpler and more casual gameplay.

Since I was the sole developer of this project, I had to be realistic about the scope and the deadline. I decided to make a mobile mini game that can be played on any device and can be completed in a few minutes. I also had to learn how to use Unity, a game engine that allows me to create 3D environments and animations. I had only three weeks to finish this project, so I had to work hard and smart.

Technologies

I am a big fan of Free Open Source Software (FOSS) and chose to use as many of them as possoble for this project

  • Godot: a cross-platform FOSS game engine that provides tools for creating 3D graphics, physics, audio, and user interface
  • gdsrcript: a dynamic scripting language similar to python that is tightly coupled with the Godot Engine for rapid development.
  • Blender: a FOSS 3D graphics app for creating models and animation
  • Git and GitHub: version control systems that allow me to track and manage the changes in my code

Challenges

Some of the challenges I faced while working on this project were:

  • Learning how to use Godot and its features, such as terrain, lighting, animation, UI and a new language gdscript
  • Optimizing the game performance and reducing the file size for mobile devices
  • Testing and debugging the game on different platforms and screen resolutions
  • Designing a fun and engaging gameplay that is not too easy or too hard
